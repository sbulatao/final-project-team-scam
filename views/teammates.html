Then on Tuesday the 26th, I got the waterproof servo, tested it, added the code, and it works. On SUNDAY of DECEMBER 1st at around 7PM of WEEK 9/10, while looking at the schematic of the PCB design and with the help of a friend, we figured out that the ESP32 was placed wrong on the PCB board and everything was flipped. I changed the pins and placed a wire from IO26 to IO16 to get an analog input for the pH Sensor on the Arduino IDE and everything worked. The OLED was displaying the pH readings, the pH sensor was working, the pumps were working with the battery, and the servo was moving. Now on to Firebase, I have researched it beforehand on November 19. With the help of YouTube: 05 Firebase Realtime Database + ESP32 | How to save sensor data & read data to control devices? and an article: ESP32: Getting Started with Firebase (Realtime Database) I was able to get the ESP32 to store data to the Firebase realtime database and for the ESP32 to read from the Firebase realtime database. I also need to think about how to send the data from the ESP32 to the server. So for the web server, I needed to initialize the Firebase admin SDK to get the database reference, then I needed a router to get the data from Firebase realtime database under the database reference called pHSensors and another router for plotting the pHSensors data. What was missing was that I also needed a web app Firebase configuration to initialize Firebase and get the data, write the data to Firebase in RealTime, and add data entries for the web. For the ESP32, I needed the Wifi, Firebase Host and Authentication, and the server url. So the ESP32 handles sending data to Firebase, and sending data to the server. The challenges that I faced was that one of the Firebase libraries was so big that it was using about 70% of the ESP32 memory, so I needed to find a smaller library. I ended up using 95% of the ESP32 memory in total. I also had 15% of dynamic memory, so if the ESP32 uses all that dynamic memory, the realtime data plot stops since I used all of it. Which then I need to turn off the ESP32 and turn it back on so that it resets lol.
